1.1 Expresiones

Cuando estamos en el modo interactivo de ML, la cosa más sencilla de interactuar con ML es crear una expresión, la cual dará como respuesta un valor y un tipo. Para entender un poco mejor a continuación un ejemplo de una expresión 

                         1+2*3;
                        
                    
                         val it = 7 : int
Ejecutar

Como resultado del ejemplo anterior se puede evidenciar que siempre que se crea una expresión, ML dará como respuesta un valor y un tipo, más conocido en inglés como (value,type). En el ejemplo anterior la expresión a sido 1 + 2 * 3, y ML ha respondido la variable it con valor 7 y que el tipo es integer. No hay que preocuparse si no se entiende hasta el momento mucho con respecto a los valores y los tipos, más adelante iremos viendo más acerca de ellos.

La variable it juega un papel importante en ML, esta recibe el valor de cualquier tipo de expresión. Una expresión debe ser seguida de un carácter que determina que ha finalizado o hasta dónde va la expresión el cual es el punto y coma.

1.2 Constantes

Como cualquier otro tipo de lenguaje, las expresiones en ML pueden estar compuestas de operadores y operandos los cuales pueden ser variables o constantes. Las constantes también tienen un tipo de dato, los cuales veremos a continuación

Integers

Los integer (enteros) son representados en ML de la misma manera que en otros lenguajes de programación con la excepción del valor mínimo que puede tener un entero, el cual es cero. Un entero es un número de 1 o más dígitos, para representar los valores negativos de un entero es necesario anteponer el símbolo ~. Los enteros también pueden ser representados como expresiones hexadecimales, donde los caracteres 0x o 0X seguidos por una cadena de dígitos representan un valor entero positivo. A continuación, se muestra un ejemplo de la respuesta de el sistema de ML para alguna expresión hexadecimal 

                         0x1234;
                        
                    
                         val it = 4660 : int
Ejecutar

Como se ve en el ejemplo, la expresión hexadecimal da como resultado su valor en decimal como entero.

Reales

Los reales son tipos de datos decimales los cuales están representados por uno o más dígitos, al igual que los enteros, su representación negativa va acompañada del carácter ~. Para los números reales se puede representar en notación científica, utilizando el carácter e o E acompañado del ~ para notación científica hacia la izquierda o sin el símbolo ~ para la notación científica hacia la derecha. 

                          3E~3;
  3.14e12
                        
                    
                          val it = 0.003 : real
  val it = 3.14E12 : real
Ejecutar

Booleanos

Los tipos de datos booleanos solamente pueden tomar dos valores True o False, ML es case-sensitive es decir que los valores de los tipos booleanos deben ser escritos en minúsculas. Un ejemplo a continuación de los tipos de valores booleanos 

                         true;
                        
                    
                         val it = true : bool
Ejecutar

Strings

Los valores de tipo string son caracteres representados por comillas dobles como “Hola mundo”. Caracteres especiales son representados por secuencia de caracteres como en el lenguaje C, donde el backslash ( \ ) sirve como un carácter de escape. La forma principal para representar caracteres especiales que pueden generar confusión de interpretación para el sistema ML son:

  • La secuencia de dos caracteres acompañado del carácter \n es usada para representar un salto de línea
  • El carácter \t es usado para hacer tab entre caracteres
  • \\ es usado para el carácter backslash
Hay otros ciertos secuencia de escape de caracteres que son comúnmente menos usados y que pueden no ser soportados por el sistema de ML A continuación un ejemplo con valores de tipo string. 

                         "A\tB\tC\n1\t2\t3\n";
                        
                    
                          A  B  C
  1  2  3
Ejecutar

1.3 Operadores Artimeticos

Los operadores aritméticos en ML son similares a los operadores en pascal o C. Existen:

  • Operadores de adición y sustracción + , -
  • Operadores de alta procedencia como los operadores de multiplicación
    ( * ), división ( div ) o módulo ( mod )
  • Operadores de negación ~
A continuación, unas expresiones y sus respuestas en el intérprete de ML: 

                         3.0 - 4.5 + 6.7;
                        
                    
                         val it = 5.2 : real
Ejecutar

Otro ejemplo sería 

                         43 div (8 mod 3) * 5;
                        
                    
                         val it = 105 : int
Ejecutar

Los tres operadores div, mod , y * son de la misma procedencia pero los paréntesis forza a realizar la operación mod primero y usa la notación hacia la izquierda por lo que se agruparía de la forma (43 div 2) * 5 (Donde 2 fue el resultado de la operación 8 mod 3)

1.4 Operadores de comparación

En programación existen 6 operadores de comparación comunes, los cuales son = , < , > , <=, => y <>, el cual este ultimo representa la expresión “diferente de”. Podemos usar estos operadores para comparar enteros, reales, caracteres, o strings con una excepción:

  • Los reales podrían no ser comparados usando = o <>. Las otras 4 comparaciones son permitidas
Veamos algunos ejemplos de comparaciones 

                         #"Z" < #"a";
                        
                    
                         val it = true : bool
Ejecutar

Como se puede notar, el resultado de la comparación es un booleano. Esto es porque se esta trabajando con caracteres, por ende, en la jerarquía del código ASCII la Z mayúscula es mayor a la a minúscula.

Otro ejemplo sería 

                         2 < 1 + 3;
                        
                    
                         val it = true : bool
Ejecutar

Aunque el resultado es un booleano, ahora hemos operado con números reales, en la cual decimos que 2 es menor a 1 más 3 que sería igual a 4. Cosa que efectivamente es cierta

1.5 Combinación lógica de valores

Los operadores comunes de lógica dan como resultado valores booleanos, los conectores son operadores como AND, OR y NOT como los de pascal o &&, || , y ! como en C. En ML se manejan valores lógicos de combinatoria llamados andalso y orelse. Como su nombre en inglés indica, esta combinación de operadores determina la manera en como procede una instrucción en ML. A continuación, se explican dichas instrucciones. 

                         1 < 2 orelse 3 > 4;
                        
                    
                         val it = true : bool
Ejecutar

ML no evalúa la segunda condición en este ejemplo debido a que la primera condición (3 > 4) da como resultado un valor booleano true, dado el caso la primera expresión hubiese dado como resultado un valor false, entraría entonces a la segunda expresión después del orelse.

Véase también la expresión andalso. 

                         1 < 2 andalso 3 > 4;
                        
                    
                         val it = false : bool
Ejecutar

En este ejemplo es necesario evaluar ambas condiciones para dar el resultado, es decir, el resultado es false debido a que la segunda condición (3 > 4) es falsa a pesar de que la primera sea verdadera (1 < 2), por ende, el resultado que arroja ML es un false

1.6 Expresión If-Then-Else

ML nos permite utilizar expresiones condicionales de la forma if E then F else G. Esta expresión lo que garantiza es que se valide si la expresión E es verdadera, entonces se ejecutara F de lo contrario se ejecutara G. Considere la siguiente expresión condicional 

                         if 1 < 2 then 3 + 4 else 5 + 6;
                        
                    
                         val it = 7 : int
Ejecutar

El resultado es 7 debido a que la expresión tiene como primera condición (1 < 2), el cual dicha condición da como resultado un true, este porque efectivamente 1 es menor que 2, como la condición es true entonces entrara al then el cual ejecuta la suma entre 3 y 4 que da como resultado el número 7.
