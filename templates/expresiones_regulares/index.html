Introducción: Expresiones regulares y ejemplos de códigos

Para desarrollar una idea del lenguaje y cómo se utiliza, consideremos la implementación de un paquete para comparar cadenas con expresiones regulares. Estructuraremos la implementación en dos módulos, una implementación de las expresiones regulares en sí mismas y una implementación de un de un algoritmo de comparación para ellas. Estos dos módulos se describen de forma concisa con las siguientes firmas

Siguiente 
                 signature REGEXP = sig
     datatype regexp =
         Zero | One | Char of char |
         Plus of regexp * regexp |
         Times of regexp * regexp |
         Star of regexp
     exception SyntaxError of string
     val parse : string -> regexp
     val format : regexp -> string
 end
 signature MATCHER = sig
     structure RegExp : REGEXP
     val accepts : RegExp.regexp -> string -> bool
 end

La firma REGEXP describe un módulo que implementa las expresiones regulares. Consiste en una descripción de la sintaxis abstracta de las expresiones regulares, junto con las operaciones para analizarlas y eliminarlas.

Siguiente 
                 signature REGEXP = sig
     datatype regexp =
         Zero | One | Char of char |

         Times of regexp * regexp |
         Star of regexp
     exception SyntaxError of string
     val parse : string -> regexp
     val format : regexp -> string
 end
 signature MATCHER = sig
     structure RegExp : REGEXP
     val accepts : RegExp.regexp -> string -> bool
 end

La firma MATCHER describe un módulo que implementa un comparador para una noción dada de expresión regular. Contiene una función acepta que, cuando se le da una expresión regular, devuelve una función que determina si esa expresión acepta o no una cadena dada. Obviamente, el comparador depende de la implementación de las expresiones regulares. Esto se expresa mediante una especificación de estructura que especifica una dependencia jerárquica de una implementación de un comparador sobre una implementación de expresiones regulares cualquier implementación de la firma MATCHER debe incluir una implementación de expresiones regulares como un módulo constituyente. Esto garantiza que el comparador es autónomo y no depende de convenciones implícitas para determinar qué implementación de expresiones regulares emplea.

La definición de la sintaxis abstracta de las expresiones regulares en la firma REGEXP toma la forma de una declaración de tipo de datos que recuerda de una gramática libre de contexto, pero que se abstrae de las cuestiones de presentación léxica (como las precedencias de los operadores, el paréntesis, las convenciones para nombrar los operadores, etc.) La sintaxis abstracta consta de seis cláusulas correspondientes a las expresiones regulares 0, 1, a, r1 + r2, r1 r2, y r∗. Las funciones funciones parse y format especifican el parser y unparser para las expresiones expresiones regulares. El analizador sintáctico toma una cadena como argumento y devuelve una expresión regular Si la cadena está mal formada, el analizador lanza la excepción SyntaxError con una cadena asociada que describe el origen del error. El toma una expresión regular y produce una cadena que se ajusta a esa expresión regular. expresión regular. En general, hay muchas cadenas que se ajustan a las mismas expresiones regulares. mismas expresiones regulares; el descompresor generalmente trata de elegir una que sea más fácil de leer

La implementación del comparador consta de dos módulos: una implementación de expresiones regulares y una implementación del comparador mismo. Una implementación de una firma se denomina estructura. La implementación del emparejador consiste en dos estructuras, una que implementa la firma REGEXP y la otra que implementa MATCHER. Por lo tanto, el paquete global se implementa mediante las siguientes dos declaraciones de estructura
